# Материалы для лекции 4

## Файлы

- [`ai_generated_pilot_code.md`](ai_generated_pilot_code.md) — предложение AI: рабочий код пилотного инкремента с архитектурно значимым решением (нормализация разнородных данных), зашитым неявно внутри функции сравнения. Используется в **блоке 1** сценария ([../script.md](../script.md)).
- [`adr_data_normalization.md`](adr_data_normalization.md) — контрпример: то же решение, принятое явно, оформленное как ADR с рассмотренными альтернативами и явной границей между источником данных и логикой сравнения. Используется в **блоках 4 и 6**.

## Как использовать на занятии

`ai_generated_pilot_code.md` показывается в начале лекции целиком, без подсветки проблемного места — аудитория должна найти скрытое решение самостоятельно (см. тайминг в сценарии). `adr_data_normalization.md` вводится позже, как ответ на вопрос «а как это можно было сделать иначе».

## Статус

Черновик, учебный пример. Код и ADR написаны ради иллюстрации архитектурной значимости решения о нормализации данных, а не как рекомендация для реализации `pe2030-air-quality-pilot`.
